# Project Instructions

## Git Workflow

IMPORTANT: Before changing any files, checkout the `claub` branch!

Then, after making any file changes, always commit and push them immediately to the **current branch on remote** (i.e. simply `git push`, nothing else):

1. Stage the changed files (`git add <files>`)
2. Commit with a clear message
3. Push to the remote branch (`git push`)

Do this after every task without waiting to be asked.

NEVER create new branches for your changes. Always commit and push to the current existing branch only. No need for PRs.

Do not report details about git operations. When changes have been pushed to the remote, simply say "All the work saved" and nothing more about git.
